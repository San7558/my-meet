1. PROJECT OVERVIEW
================================================================================

AI Translation Room is a professional SaaS-style web application that provides
real-time multilingual speech translation with AI-powered gesture safety
monitoring.

Users sign in with Google, join a live session, speak in any language, and
instantly see translated captions in their chosen target language. The system
also monitors for inappropriate hand gestures and enforces a 3-strike
warning system.

After a session, users can download the complete translated transcript as a
professionally formatted PDF document.


================================================================================
2. WHAT PROBLEM THIS PROJECT SOLVES
================================================================================

In multilingual meetings, conferences, or classrooms:

- Language barriers prevent real-time understanding.
- Manual translation is slow and error-prone.
- There is no automated way to detect inappropriate gestures.
- Transcripts are often lost after the session ends.

AI Translation Room solves all of these:

- Automatic speech-to-text in any language (Deepgram).
- Automatic translation to the user's chosen language (LibreTranslate).
- AI gesture detection for safety moderation (MediaPipe).
- One-click PDF export of the full session transcript (jsPDF).

3. FULL FEATURE EXPLANATION
================================================================================

3.1 Landing Page
    - Premium dark SaaS design.
    - Glassmorphism cards.
    - 3D rotating globe (React Three Fiber).
    - Feature highlights, workflow steps, tech stack display.
    - Call-to-action button leading to login.

3.2 Authentication
    - Google Sign-In using Firebase Authentication.
    - Protected routes redirect unauthenticated users to login.
    - User profile (name, email, avatar) displayed on dashboard.

3.3 Dashboard
    - User profile card with online status.
    - Target language selector (English, Tamil, Hindi, Telugu, Malayalam, Kannada).
    - "Start Translation Session" button navigates to session page.

3.4 Session Page
    - Camera and microphone permission request on entry.
    - Live camera preview with LIVE badge.
    - Camera On/Off and Mic On/Off toggle buttons.
    - Deepgram real-time speech-to-text (starts when mic is turned on).
    - Translated captions appear in the description box.
    - MediaPipe hand gesture detection runs automatically.
    - Gesture warning counter badge (top-right corner).
    - Stop Session button ends all streams.
    - PDF download button exports the transcript.

3.5 Gesture Detection
    - Uses MediaPipe Hands to track 21 hand landmarks.
    - Detects middle-finger gesture (middle extended, others folded).
    - 2500ms cooldown between detections to prevent spam.

3.6 Warning System
    - Warning 1/3 → Yellow badge, red alert in description box.
    - Warning 2/3 → Yellow badge, red alert in description box.
    - Warning 3/3 → Red pulsing badge, session auto-terminates.
    - Each violation is logged to MongoDB via the backend API.

3.7 PDF Export
    - Uses jsPDF to generate a professional A4 PDF.
    - Includes: title, user name, date/time, language, all transcript lines.
    - System alert lines appear in red bold.
    - Handles long text wrapping and page overflow.
    - Footer with page numbers on every page.

3.8 Translation
    - Uses LibreTranslate (self-hosted or public instance).
    - Automatic source language detection (source: "auto").
    - Does NOT use Google Translate API.

7. STEP-BY-STEP LOCAL SETUP
================================================================================

PREREQUISITES:
  - Node.js 18 or later installed.
  - Docker Desktop installed and running.
  - A web browser (Chrome recommended).
  - A Google account (for Firebase authentication).

STEPS:

Step 1: Clone or download the project.
  git clone <your-repo-url>
  cd my-meet

Step 2: Start LibreTranslate via Docker.
  docker run -it -p 5001:5000 libretranslate/libretranslate
  Wait until it says "Running on http://0.0.0.0:5000".
  It will be accessible at http://localhost:5001 on your machine.

Step 3: Setup the backend.
  cd server
  cp .env.example .env
  Open .env and fill in your values (see sections 8–14 below).
  npm install
  npm run dev
  You should see:
    ✅ MongoDB connected
    ✅ Firebase Admin initialized
    🚀 Server listening on http://localhost:5000

Step 4: Setup the frontend.
  cd client
  cp .env.example .env
  Open .env and fill in your values (see sections 8–14 below).
  npm install
  npm run dev
  Frontend opens at http://localhost:5173.

Step 5: Open http://localhost:5173 in Chrome.

6. Paste this into server/.env as MONGODB_URI.

15. HOW TO RUN LIBRETRANSLATE USING DOCKER
================================================================================

LibreTranslate is a free, open-source translation API.

REQUIREMENTS:
  - Docker Desktop must be installed and running.

COMMAND:
  docker run -it -p 5001:5000 libretranslate/libretranslate

WHAT HAPPENS:
  - Docker downloads the LibreTranslate image (first time only).
  - It downloads language models (~1-2 GB, takes a few minutes).
  - When ready, it outputs: "Running on http://0.0.0.0:5000"
  - On your machine, it's accessible at: http://localhost:5001

TEST:
  Open http://localhost:5001 in your browser.
  You should see the LibreTranslate web interface.

IMPORTANT:
  Keep this terminal window open while using the app.
  If you close it, translation will stop working.

16. HOW AUTO LANGUAGE DETECTION WORKS
================================================================================

When you speak into the microphone:

1. Deepgram captures the audio via MediaRecorder.
2. Audio chunks are sent every 250ms via WebSocket to Deepgram servers.
3. Deepgram's Nova-2 model (with language=multi) automatically detects
   the spoken language and transcribes it to text.
4. The transcript is sent to the backend /api/translate endpoint.
5. The backend forwards it to LibreTranslate with source: "auto".
6. LibreTranslate auto-detects the source language and translates it
   to the user's chosen target language.
7. The translated text appears in the description box.

This means users can speak in ANY supported language and see translations
without manually selecting their source language.


================================================================================
17. HOW TRANSLATION WORKS
================================================================================

FLOW:
  User speaks → Deepgram STT → transcript text
  → Frontend api.js sends POST /api/translate with Bearer token
  → Backend controller validates text + targetLanguage
  → Backend looks up language code from languageMap
  → Backend service calls LibreTranslate /translate
  → LibreTranslate returns { translatedText: "..." }
  → Backend returns { translatedText: "..." } to frontend
  → Frontend displays "User 1: <translated text>" in DescriptionBox

IMPORTANT:
  - The app uses LibreTranslate ONLY.
  - Google Translate API is NOT used.
  - All translation requests go through the backend (never direct from frontend).
  - Firebase Bearer token is required for the /api/translate endpoint.

18. HOW GESTURE DETECTION WORKS
================================================================================

MediaPipe Hands is loaded via CDN scripts in index.html.
It tracks 21 hand landmarks in real-time from the camera feed.

DETECTION LOGIC:
  Middle finger extended: landmarks[12].y < landmarks[9].y
  Index finger folded:    landmarks[8].y  > landmarks[6].y
  Ring finger folded:     landmarks[16].y > landmarks[14].y
  Pinky finger folded:    landmarks[20].y > landmarks[18].y

  If ALL four conditions are true → middle-finger gesture detected.

COOLDOWN:
  After a detection, there is a 2500ms cooldown before the next
  detection can trigger. This prevents rapid repeated warnings.

PROCESSING:
  - Runs entirely in the browser (no video sent to backend).
  - Uses requestAnimationFrame for continuous frame processing.
  - Starts automatically when the camera stream is available.


================================================================================
19. HOW THE WARNING SYSTEM WORKS
================================================================================

When a middle-finger gesture is detected:

  Warning 1/3:
    - Badge turns yellow.
    - Red alert line appears in description box.
    - Violation logged to MongoDB via POST /api/violations.

  Warning 2/3:
    - Badge stays yellow.
    - Another red alert line appears.
    - Another violation logged.

  Warning 3/3:
    - Badge turns red and pulses.
    - Final red alert line: "User 1 was removed from the session."
    - Session auto-terminates (Deepgram, MediaPipe, camera all stop).
    - Controls are disabled.
    - User can only go "Back to Dashboard" or download PDF.

20. HOW PDF EXPORT WORKS
================================================================================

The PDF button is in the session page action bar.
It is disabled when there are no transcript lines.

When clicked, it generates a PDF using jsPDF:

  1. Creates an A4 portrait document.
  2. Adds a title: "AI Translation Room — Session Transcript".
  3. Adds metadata: user name, date/time, selected language, line count.
  4. Adds a horizontal divider.
  5. Adds all transcript lines:
     - Normal lines in dark text.
     - System Alert lines (starting with ⚠) in red bold.
  6. Handles long text wrapping to page width.
  7. Automatically adds new pages when content overflows.
  8. Adds a footer on every page: "AI Translation Room — Page X of Y".
  9. Saves the file as "session-transcript.pdf".

25. COMMON ERRORS AND FIXES
================================================================================

ERROR: "MONGODB_URI not defined" or "MongoDB connection failed"
FIX:   Check server/.env has a valid MONGODB_URI.
       Ensure your IP is whitelisted in MongoDB Atlas.
       Ensure username and password are correct.

ERROR: "Firebase Admin initialization failed"
FIX:   Check FIREBASE_PROJECT_ID, FIREBASE_CLIENT_EMAIL, FIREBASE_PRIVATE_KEY
       in server/.env. Wrap FIREBASE_PRIVATE_KEY in double quotes.

ERROR: "Unauthorized: No token provided"
FIX:   The frontend must send Authorization: Bearer <token> header.
       Ensure the user is logged in before making API requests.

ERROR: "LibreTranslate error" or translation not working
FIX:   Ensure Docker is running and LibreTranslate is accessible at
       http://localhost:5001. Try opening it in your browser.

ERROR: "VITE_DEEPGRAM_API_KEY is not set" or Deepgram not connecting
FIX:   Check client/.env has VITE_DEEPGRAM_API_KEY set.
       Ensure the API key is valid at https://console.deepgram.com.

ERROR: Camera/microphone permission denied
FIX:   Click the lock icon in Chrome's address bar → allow camera and mic.
       Or go to chrome://settings/content and reset permissions.

ERROR: MediaPipe Hands not loading
FIX:   Ensure CDN scripts are in client/index.html.
       Check browser console for network errors loading MediaPipe files.
       Ensure you have internet access (CDN scripts load from jsdelivr).

ERROR: PDF button is disabled
FIX:   The PDF button only activates when there are transcript lines.
       Speak into the microphone first to generate transcript content.

ERROR: "Module not found" or import errors
FIX:   Run npm install in both client/ and server/ directories.
       Check that import paths use correct relative paths.

ERROR: CORS errors in browser console
FIX:   Ensure CLIENT_URL in server/.env matches the frontend URL.
       For local development: CLIENT_URL=http://localhost:5173
